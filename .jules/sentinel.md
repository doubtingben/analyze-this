## 2024-04-06 - XSS in Metrics and Notes Rendering
**Vulnerability:** User-controlled strings (e.g. `config.label` and `error.message`) were directly injected into `innerHTML` using template literals in `backend/static/app.js` (lines 987, 1066, 2268), posing a DOM-based Cross-Site Scripting (XSS) risk.
**Learning:** In vanilla JS apps, injecting data into `innerHTML` via template strings is a common vector. While `textContent` is preferred, when mixing tags, a helper like `escapeHtml()` must be explicitly called on the variables before interpolation.
**Prevention:** Always use `escapeHtml()` or `textContent` for dynamic values inside DOM components rendering logic, especially error messages or status labels derived from APIs.

## 2024-04-29 - SSRF Vulnerability in Podcast Fetching
**Vulnerability:** The function `_extract_remote_text_with_diagnostics` in `backend/podcast_content.py` fetched remote URLs using `requests.get(allow_redirects=True)` without validating the IP addresses. This allowed Server-Side Request Forgery (SSRF), permitting attackers to potentially access internal metadata endpoints (e.g., `169.254.169.254` on AWS) or private network resources.
**Learning:** Even if a requested URL appears external, HTTP redirects can transparently bounce the server connection to internal private IPs. `requests.get` follows these by default. To securely handle user-provided URLs in Python without full custom transports, one must manually implement a redirect loop (`allow_redirects=False`), extract the `Location` header, securely parse it, resolve the hostname, and explicitly check if the IP is private/loopback/link-local/unspecified before making each subsequent request. Python's `socket.gethostbyname` followed by checking with `ipaddress` provides basic protection, though it may still be susceptible to TOCTOU DNS Rebinding attacks.
**Prevention:** Never use `allow_redirects=True` with user-supplied URLs if fetching them internally. Implement a manual redirect loop that parses the `Location` header, resolves its hostname, and explicitly blocks loopback, private, link-local, and `0.0.0.0` IP addresses using the `ipaddress` module before issuing the next `requests.get(allow_redirects=False)`.
